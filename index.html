Concept Explanation

Definition: Automatic Repeat Request (ARQ) is an error-control strategy for data transmission that relies on acknowledgments (ACKs) and timeouts to achieve reliable data delivery over an unreliable service.

Process Steps:

  • Sender transmits a packet and starts a local timer.
  • Receiver checks for errors and sends an ACK if data is valid.
  • If ACK arrives before timeout, sender moves the window.
  • If timeout occurs, sender retransmits the unacknowledged frame.
