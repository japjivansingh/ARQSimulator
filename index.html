Selective Repeat & Stop-and-Wait ARQ

Definition: Automatic Repeat Request (ARQ) is an error-control strategy for data transmission that relies on acknowledgments (ACKs) and timeouts to achieve reliable data delivery over an unreliable service[cite: 6, 9].

Process Steps[cite: 7]:

  • Sender transmits a packet and starts a local timer[cite: 7].
  • Receiver checks for errors and sends an ACK if data is valid[cite: 7, 8].
  • If ACK arrives before timeout, sender moves the window[cite: 7, 8, 11].
  • If timeout occurs, sender retransmits the unacknowledged frame[cite: 7, 11].
